Object Oriented Programming Features

Object-Oriented Programming is a programming paradigm based on the creation of reusable “objects” that have their own properties and behavior that can be acted upon, manipulated, and bundled. These objects package related data and behaviors into representations of real-life objects. OOP is a widely used paradigm across various popular programming languages like Python, C++, and Java. Many developers use OOP because it makes your code reusable and logical, and it makes inheritance easier to implement. It follows the DRY principle, which makes programs much more efficient.

Polymorphism

The ability of object-oriented programming to use a common interface for multiple forms and data types is known as polymorphism. Polymorphism is an ability (in OOP) to use a common interface for multiple forms (data types). Suppose, we need to color a shape, there are multiple shape options (rectangle, square, circle). However, we could use the same method to color any shape. This concept is called Polymorphism.

For Example : In our calculator program, we have added the getoutput(self), method which implements the polymorphism feature. In this project, we are performing different operation like addition, subtraction, multiplication and division and same method i.e getoutput(self) is returning values for the operation performed. The names of both methods are the same, here, but the purpose is different. Next, a string value will be taken from the user to create an object and to call the method based on the value.

Abstraction

Abstraction in python is defined as a process of handling complexity by hiding unnecessary information from the user. This is one of the core concepts of object-oriented programming (OOP) languages. That enables the user to implement even more complex logic on top of the provided abstraction without understanding or even thinking about all the hidden background/back-end complexity. A powerful way to manage abstraction is through the use of hierarchical classification. This allows us to layer the semantics of complex systems, breaking them into more manageable pieces.

For Example: In the example there is an abstract method add_operation_to_history(). There are four child classes additioncalc_project, divisioncalc_project, subtractioncalc_project and multiplicationcalc_project derived from Calculation_project that implement the abstract method add_operation_to_history() as per their functionality. As a user we are abstracted from that implementation when an object of individual class is created and add_operation_to_history() method is invoked using that object. In this way, we have implemented abstraction feature of OOP concept in our project.

Real Life Examples

The real world examples for the OOP programming features are as follows :


1) Abstraction: The feature of Abstraction can be seen in the following image, which has manual and a television set. In this case, the user is just provided with the manual to set up the television set with a set of instruction that needs to be followed in order to set up the television. In this abstraction feature, the internal abstract case of setup guide is called and user is never aware of it. This illustrates the real life example of the abstraction feature of OOP concepts.

2) Encapsulation: The Encapsulation feature hides the internal logic use in implementing the ATM card. When a user uses an ATM functionality, like withdraw or depositing the cash, he is not aware of internal logic use for implementing this feature. This illustrates that while working with classes and dealing with sensitive data, providing global access to all the variables used within the program is not a good choice. Thus, Encapsulation offers a way for us to access the required variables without providing the program full-fledged access to any of those variables which is use in our example of the ATM function which illustrates the encapsulation feature of the OOP concepts.

3) Inheritance: The following example illustrates the inheritance feature of the OOP. The child is derived from the parent, where child resembles the functionality of the parent along with inheriting their properties and characteristics. It represents real-world relationships well.It is transitive in nature, which means that if class B inherits from another class A, then all the subclasses of B would automatically inherit from class A. Also, it allows us to add more features to a class without modifying it.

4) Polymorphism : The given example of microwave implements the feature of the polymorphism. The same microwave can be use for preparing the cake, or just steaming the rice. There is no need for having separate microwave for performing the separate operation. This illustrates the function of polymorphism in the real life world. In the programming world, Polymorphism refers to the ability of the function with the same name to carry different functionality altogether. It creates a structure that can use many forms of objects.
